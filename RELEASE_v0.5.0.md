科研龙虾从"能用"走向"好用" —— 文献库补齐了 Zotero/EndNote 一键导入, 监控系统可以自由创建任意数量的定时任务, 工作区不再被系统文件污染, 任务面板新增甘特图和智能提醒。

## 四大更新

### 1. 文献库: 从"书架"升级为"图书馆"

**之前的问题**: 论文读完后会从界面消失; 只能手动一篇篇添加; 导出的 BibTeX 字段不全。

**现在**:

- **三栏浏览**: 收件箱 (待处理) / 归档 (已读) / 收藏 (打星) —— 论文不再凭空消失
- **一键导入本地文献库**: 支持 Zotero 和 EndNote, 自动读取你电脑上已有的论文数据, 无需手动录入
- **通用格式支持**: 新增 RIS 格式导入 (几乎所有文献管理器都能导出 RIS), BibTeX 导出修复为完整学术格式
- **更丰富的论文信息**: 新增关键词、语言、论文类型、卷期页码、出版商、学科分类、被引次数共 11 个字段
- **全文搜索增强**: 搜索范围扩展到关键词字段

### 2. 监控系统: 从"5 个固定闹钟"升级为"自由创建的雷达网"

**之前的问题**: 只有 5 个写死的定时任务 (arXiv 扫描、截止日提醒等), 不能新增也不能自定义。

**现在**:

- **无限监控器**: 可以创建任意数量的监控任务, 每个都能独立设置数据源、关键词、时间表
- **7 种预设模板**: arXiv 日扫、引用追踪、截止日提醒、组会准备、周报、GitHub Release 监控、网页变动监控
- **灵活调度**: 每个监控器独立开关、独立排期, 互不干扰
- **面板重新设计**: 从旧的"雷达"面板升级为全新的"监控"面板, 更直观

### 3. 工作区: 从"配置目录"回归"科研项目文件夹"

**之前的问题**: 打开工作区文件树, 映入眼帘的是 AGENTS.md、SOUL.md 等 8 个系统文件, 用户一脸懵。

**现在**:

- **系统文件搬家**: 8 个提示词文件自动迁移到隐藏的 `.ResearchClaw/` 目录, 工作区恢复清爽
- **右键新建**: 终于可以像 VS Code 一样右键 → 新建文件 / 新建文件夹了 (支持中文文件名)
- **版本历史增强**: 支持加载更多历史记录 + 点击查看每次改动的具体差异
- **高级用户**: 可在设置中切换显示/隐藏系统文件

### 4. 任务面板: 从"待办清单"升级为"项目管理"

**之前的问题**: 只能看列表, 无法直观感知时间线; 截止日临近了也没提醒。

**现在**:

- **甘特图**: 点击按钮即可查看所有任务的时间线视图, 一目了然
- **智能提醒**: 4 级自适应提醒 (静默 → 温和 → 紧急 → 严重), 越临近截止日提醒越频繁
- **优先级快速切换**: 下拉菜单直接改优先级, 不用进详情
- **流水线阶段**: 每个任务可标记属于科研流水线的哪个阶段

## 升级方式

**本地安装用户**:
```bash
curl -fsSL https://wentor.ai/install.sh | bash
```

**Docker 用户**:
```bash
docker pull ghcr.io/wentorai/research-claw:0.5.0
```

升级是无感的 —— 你的论文数据、任务、工作区文件全部自动保留, 数据库会在首次启动时自动完成 3 步增量迁移 (v7→v8→v9), 工作区系统文件会自动搬到 `.ResearchClaw/` 目录。

## 下集预告: v0.6.0 — 大扫除

v0.5.0 新增了大量功能, 但科研龙虾的"武器库"也变得庞杂了: **41 个工具、82 个接口、432 个技能**。不是每一个都还能正常工作, 也不是每一个都值得保留。

v0.6.0 的核心任务是**全面盘点和清理**:

- **文献 API 有效性验证**: 逐个测试 Semantic Scholar、arXiv、OpenAlex、CrossRef、PubMed、Unpaywall 六大学术 API 的连通性和返回质量, 标记失效/降级的数据源, 建立学科→数据源路由表
- **工具清单审计**: 41 个 Agent 工具逐一验证, 确认参数是否正确、返回值是否被 Dashboard 消费、是否有僵尸工具
- **技能有效性梳理**: 432 个 SKILL.md 按学科分类抽检, 清理过时内容、修复格式错误、补充缺失的关键词索引
- **Bootstrap 流程升级**: 首次使用时自动检测本地环境 (Python、LaTeX、Playwright), 一键安装缺失的科研工具链
- **搜到即入库**: 搜索论文后自动加入文献库, 不再需要手动确认

目标: 让每一个工具都确实能用, 每一个技能都确实有效, 每一个 API 都确实通畅。
